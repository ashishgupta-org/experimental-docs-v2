title: Benefits of AI for Service (XO11)
---

AI for Service is a major evolution of the Kore.ai platform. It addresses the limitations of fragmented conversational experiences and introduces advanced AI capabilities through a unified, intelligent, and collaborative platform.

## Automation AI

### Unified Conversation Session

Earlier, integrations with Kore.ai products—Platform, SmartAssist, Search Assist, and Agent Assist—relied on APIs or Webhooks. This resulted in disjointed conversations with no central session to track complete interactions.

With AI for Service, conversations are unified across modules, enabling seamless end-to-end tracking within a single session. This improves analytics, simplifies cross-module data retrieval, and enhances workflow efficiency.

### Improved Dialog Builder Experience

The new Dialog Builder v3 offers an intuitive, collaborative environment with an infinite canvas, multi-user collaboration, visual flow design, and no-code templates. This reduces development time and improves the quality and maintainability of conversational flows.

**Key capabilities:**

- **Free Flow Designing**: Design flows without defining logic upfront—similar to tools like Lucidchart or Miro.
- **Easy Transitions**: Connect nodes without fully defining transition rules. Start with a description and enforce rules later during testing or publishing.
- **Infinite Canvas**: Arrange flows logically by purpose or objective for easy review and auditing.
- **Commenting**: Add comment threads to ask questions and exchange ideas.
- **Spacious Nodes**: Read and write key information—name, type, prompts/messages, and transition conditions—clearly.
- **Annotation Tools / Whiteboarding**: Add annotations to explain flows, capture ideas, and support brainstorming alongside the flow.
- **Co-Designing**: Allow multiple users to collaborate in parallel—essential when conversation designers and AI Agent developers co-build.
- **Sharing**: Share designs with external stakeholders to collect feedback or get approvals.
- **New Node Panel**: Simplified panel with four broad categories: Inform, Ask, Action, and AI.
- **No-Code Templates**: Configure templates in the UI without writing code.
- **Node Sequences**: Group nodes into sequences for better flow organization.

### Advanced AI Capabilities

DialogGPT and enhanced RAG (Retrieval-Augmented Generation) capabilities enable zero-training intent detection, contextual understanding, and intelligent conversation management. The Agent Node with tool calling enables sophisticated LLM-powered interactions that handle complex tasks and integrate with external systems.

### DialogGPT

DialogGPT introduces a paradigm shift in intent detection and conversation management. It uses embedding-based retrieval combined with generative AI to improve intent detection and conversation handling—without requiring training utterances.

**Key capabilities:**

- Contextual intent detection without training utterances
- Identification and fulfillment of ambiguous and multi-intents
- Management of common conversation intents: pausing, repeating, restarting, and transferring to agents
- Granular intent identification using the knowledge graph, with LLM-powered clarifying questions
- Out-of-the-box event handlers to fulfill user requests

With DialogGPT, businesses deliver more accurate, responsive, and efficient user experiences while simplifying development.

### Seamless Integration with Search Module (RAG Capabilities)

Powered by LLMs, Search AI provides accurate, context-aware answers. It integrates diverse data sources—web crawlers, structured data, documents, catalogues, and knowledge repositories—into a comprehensive answer index.

**Key capabilities:**

- Answer any question from the knowledge base
- Connect to 50+ external connectors for automatic knowledge base sync
- Customize the chunking strategy
- Manage the RAG pipeline
- Test answers using the debug window

### Agent Node with Tool Calling

The Agent Node uses LLMs and generative AI with tool calling to create versatile AI agents capable of handling complex, dynamic, data-driven tasks.

**Key capabilities:**

- **Entity Collection**: Simplifies gathering entities within a conversation, reducing the need for multiple entity nodes and making interactions more natural.
- **System Context, Business Rules, and Exit Scenarios**: Incorporates system context, business rules, and predefined exit scenarios to guide conversations and maintain alignment with enterprise rules.
- **Multilingual Support**: Supports English and non-English languages for diverse user bases.
- **Configuration Flexibility**: Configurable like any other node, enabling seamless integration into existing conversational flows.
- **Interaction with External Systems**: Tool calling enables real-time data retrieval, calculations, and system-specific operations for dynamic, data-driven experiences.
- **Dynamic Prompt Enhancement**: The prompt dynamically adapts to include tool definitions and contextual information so the model responds or calls a tool as appropriate at each step.

### Seamless Integration with Agentic Apps for Multi-agent Orchestration

Fully autonomous Agentic Apps integrate easily with the Platform, simplifying the creation of highly contextual, self-service automation experiences using multi-agent orchestration.

**Key features:**

- Choice between **Orchestrated Autonomy** or **Full Autonomy** automation modes
- Full support for digital and voice channels

Features like Experience Flows for channel-specific welcome experiences, conditional FAQ answers, and an enhanced Marketplace with ready-to-use templates accelerate time-to-value and reduce the complexity of managing conversational AI applications.

## Search AI

Search AI is the next generation of Kore.ai's search and retrieval platform, with improvements across data ingestion, content extraction, retrieval, and answer generation.

### Intelligent and More Accurate Answer Generation

**Agentic RAG Architecture**

Search AI uses an Agentic RAG (Retrieval-Augmented Generation) architecture where multiple autonomous agents collaborate to optimize query interpretation, retrieval, and response generation. This delivers highly accurate, context-aware answers for complex and multi-faceted queries. [Learn more](/ai-for-service/searchai/agentic-rag-and-business-rules).

**Advanced Retrieval Technology**

Search AI offers hybrid retrieval based on RAG Fusion, combining multiple keyword and vector scores to boost accuracy, relevance, and performance. [Learn more](/ai-for-service/searchai/agentic-rag-and-business-rules).

**Channel-Aware Response Formatting**

Responses are automatically optimized for voice or digital channels, ensuring the most effective format for each medium.

**Contextual Answer Generation**

Search AI improves response accuracy by sending entire documents—rather than fragments—as context to LLMs, enabling the model to grasp structure and intent for more relevant answers.

### Improved Content Ingestion Capabilities

**6x More Connectivity Options**

- 60+ connectors—a 6x increase over SearchAssist
- Support for both structured and unstructured data from connectors
- Universal RACL support across connectors
- Streamlined configuration and setup

[Learn more](/ai-for-service/searchai/content-sources#connectors).

**Superior Web Crawling**

- New, improved crawler with significantly higher success rates
- 80% improvement in web crawling speed
- Better handling of complex website structures, dynamic content, and JavaScript-heavy pages

[Learn more](/ai-for-service/searchai/content-sources#websites).

**Flexible Field Mapping for Content Ingestion**

- Map fields from source applications to the Search AI schema for consistency and relevance
- Use post-processor scripts to update content received from source applications
- Test and save field mappings before applying them for ingestion

### Advanced Content Extraction and Processing

**Flexible Extraction Framework**

Search AI replaces SearchAssist's static approach with a customizable extraction strategy framework that adapts to different document types:

- [**Image-based Document Extraction**](/ai-for-service/searchai/index-configuration#image-based-document-extraction): Uses VDR-based embeddings to capture semantic meaning from text and images, improving accuracy for complex documents with nontextual layouts (tables, forms).
- [**Markdown Extraction**](/ai-for-service/searchai/index-configuration#markdown-extraction): Transforms each page into structured Markdown before processing, preserving semantic structure.
- [**Layout-Aware Extraction**](/ai-for-service/searchai/index-configuration#layout-aware-extraction): Combines OCR, layout detection models, and layout awareness rules to comprehend tables, graphs, and charts.
- [**Advanced HTML Extraction**](/ai-for-service/searchai/index-configuration#advanced-html-extraction): Intelligent processing of images and tables from HTML documents.
- [**Custom Extraction**](/ai-for-service/searchai/index-configuration#custom-extraction): Allows enterprises to insert custom extraction logic into the search data pipeline.

**Multimodal Content Support**

Search AI supports extraction and processing of diverse content types—text, tables, images, and layout-based visual structures—using different extraction techniques. Users can define different strategies for different content types.

**Custom Content Transformation During Extraction**

The content transformation layer enriches content after extraction and before vectorization. It now includes an API stage for custom processing and supports JavaScript for enrichment logic (replacing painless scripts), making the process more developer-friendly. [Learn more](/ai-for-service/searchai/index-configuration#content-enrichment-using-workbench).

**Chunk-Level Processing**

The Chunk Workbench provides granular control over how content is transformed at the individual chunk level—for example, appending page summaries to each chunk for enhanced context. [Learn more](/ai-for-service/searchai/index-configuration#content-enrichment-using-workbench).

**Multi-Vector Generation**

Search AI supports associating multiple embedding vectors with a single chunk, improving retrieval accuracy by capturing different important fields. Each vector can be weighted by the relative importance of its fields during search and retrieval. [Learn more](/ai-for-service/searchai/retrieval-answer-configuration).

**Enhanced Embedding Models**

- Support for E5 and VDR (Image Embeddings) models for superior semantic understanding across text and image data
- Custom embedding support without requiring middleware
