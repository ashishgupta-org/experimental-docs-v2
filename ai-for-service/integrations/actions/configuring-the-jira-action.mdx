## Authorizations Supported

The XO Platform supports basic authentication for JIRA. See [App Authorization Overview](/ai-for-service/app-settings#authorization-profiles) for details.

<Note>JIRA integration can only be authorized for Developer and Enterprise editions — not Trial. See [JIRA documentation](https://developer.atlassian.com/cloud/jira/platform/oauth-2-3lo-apps/#overview) for account types.</Note>

| Authorization Type | Basic OAuth |
|---|---|
| Pre-authorize the Integration | Yes |
| Allow Users to Authorize the Integration | Yes |

## Step 1: Enable the JIRA Action

**Prerequisites:**

- Create a [developer account](https://developer.atlassian.com/) in JIRA. See [JIRA Help Center](https://developer.atlassian.com/cloud/jira/platform/oauth-2-3lo-apps/#overview) for details.
- Copy the Client ID, Client Secret Key, Authorization URL, and Callback URL.

**Steps:**

1. Go to **App Settings** > **Integrations** > **Actions**.
2. Select the **JIRA** action in the **Available** region.

### Pre-authorize the Integration

**System Authorization**

1. Go to **App Settings** > **Integrations** > **Actions** and select **JIRA**.
2. In **Configurations**, select the **Authorization** tab.
3. Set **Authorization Type** to **Pre-authorize the Integration** > **OAuth**.

   ![Authorization Type-OAuth](/ai-for-service/integrations/actions/images/jira-action-img2.png)

4. Select the **System** card and click **Authorize**.

   ![System card](/ai-for-service/integrations/actions/images/jira-action-img3.png)

5. You are redirected to `https://developer.atlassian.com/console/`. Enter your credentials and click **Allow Access**.

**Custom Authorization**

1. Select **Custom** and click **Select Authorization** > **Create New**.

   ![Create new auth](/ai-for-service/integrations/actions/images/jira-action-img4.png)

2. Select **OAuth v2**. See [Setting Up Authorization Using OAuth v2](/ai-for-service/app-settings/dev-tools/bot-authorization/setting-up-authorization-using-oauth-v2).
3. Enter the OAuth v2 credentials:
   - Call back URL
   - Identity Provider Name
   - Client ID
   - Client Secret
   - Authorization URL
   - Token Request URL
   - Scope
   - Refresh Token URL

   ![Authorization credentials](/ai-for-service/integrations/actions/images/jira-action-img5.png)

4. Click **Save Auth**, then select the new profile.

   ![Authorization profile](/ai-for-service/integrations/actions/images/jira-action-img6.png)

5. Click **Authorize**, enter credentials at `https://developer.atlassian.com/console/`, and click **Allow Access**.
6. Click **Enable**.

   ![Enable integration](/ai-for-service/integrations/actions/images/jira-action-img7.png)

<Note>The JIRA action moves from _Available_ to _Configured_ after enabling.</Note>

### Allow End User to Authorize

1. Go to **App Settings** > **Integrations** > **Actions** and select **JIRA**.
2. In **Configurations**, select the **Authorization** tab.
3. Set **Authorization Type** to **Allow Users to Authorize the Integration** > **OAuth**.

   ![Authorization Type-OAuth](/ai-for-service/integrations/actions/images/jira-action-img8.png)

**System Authorization**

1. Select the **System** card.

   ![System card](/ai-for-service/integrations/actions/images/jira-action-img9.png)

2. Click **Enable**. A link is sent to the end user to authorize.
3. The user clicks the link and enters credentials to authorize.

**Custom Authorization**

1. Select **Custom** and click **Select Authorization** > **Create New**, following the Custom Authorization steps above.
2. You can also select an existing profile:

   ![Select existing auth profile](/ai-for-service/integrations/actions/images/jira-action-img10.png)

3. Click **Enable**.

## Step 2: Install the Jira Action Templates

1. On the **Integration Successful** dialog, click **Explore Templates**.

   ![Explore templates](/ai-for-service/integrations/actions/images/jira-action-img11.png)

2. Click **Install** to begin installation.

   ![Begin installation](/ai-for-service/integrations/actions/images/jira-action-img12.png)

3. Once installed, click **Go to Dialog**. A dialog task for each template is auto-created.

   ![Dialog task for each installed template](/ai-for-service/integrations/actions/images/jira-action-img13.png)

4. Select the desired dialog task and click **Proceed**.

   ![Select desired dialog task](/ai-for-service/integrations/actions/images/jira-action-img14-tem-img2.png)

5. The dialog task is auto-created and the canvas opens with all required entity nodes, service nodes, and message scripts.
