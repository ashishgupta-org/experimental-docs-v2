This document explains the integration of **OmniChannel Chat (Messaging)** with **Agent AI** in Salesforce. This integration delivers AI-powered suggestions, automated responses, and conversation summaries directly in the Salesforce console. It helps agents respond faster, improve accuracy, and maintain consistent communication.

## Prerequisites

* Enabled OmniChannel Chat
* Ability to complete chat conversations in Salesforce

## Steps to Configure OmniChannel Chat

1. Install the Kore [Agent AI package](https://appexchange.salesforce.com/appxListingDetail?listingId=a0N4V00000HSGlnUAH) in the org.
2. Configure the Agent AI package and features:
    1. Click **App Launcher**.
    ![app-launcher](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/app-launcher-1.png)

    2. Type **kore** in the search bar, and select **Kore Agent AI Feature Configuration** from the displayed list.
    ![agentai-feature-configuration](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/agentai-feature-configuration-2.png)

    3. Click **New** from the right-top corner of the page.
    4. Enter **Omnichannel Configuration** in the **Configuration Name** field. (You can enter any relevant name.)
    ![omnichannel-configuration](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/omnichannel-configuration-3.png)

    5. Keep the **CTI Provider** field as is. For an Omnichannel solution, selecting a CTI Provider is not required.
    6. Select **Chat** or **Chat and Voice** from the **Salesforce OmniChannel Solution** list.
    ![salesforce-omnichannel-solution](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/salesforce-omnichannel-solution-4.png)

    7. Select **Active**.
    8. Click **Save**.

3. Configure Salesforce with Kore Agent AI credentials:
    1. Click **App Launcher**.
    2. Type **config** in the **search bar,** and select **Kore Agent AI Configuration** from the displayed list.
    3. Click **New** from the right-top corner of the page.
    4. Enter the required values in the **Agent AI URL**, **Bot ID**, **Client ID**, and **Client Secret** fields. To get these values:
        1. Sign in to [AI for Service](https://platform.kore.ai/).
        2. Go to **Flows & Channels** > **Channels**.
        3. Click **Digital** > **Web/Mobile Client**.
        4. Click to expand the **JWT App Details** section.
        5. Click the **Copy** buttons against the required fields.
        ![web-mobile-client](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/web-mobile-client-5.png)

    5. Select **Active**.
    6. Click **Save**.
    ![kore-agent-ai-configuration](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/kore-agent-ai-configuration-6.png)

## Add Component to the page

1. Go to **App Launcher** and select **Chat Transcripts**. (This is required for old legacy chat.)
![chat-transcripts](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/chat-transcripts-7.png)

2. Open any record of chat transcripts.

    <Note>For new Salesforce chat (Salesforce in-app and web messaging), conversations are stored inside the **Messaging Sessions** page. Open **Messaging Sessions** records.</Note>

    ![messaging-sessions-record](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/messaging-sessions-record-8.png)

3. Edit the page by clicking the **gear** icon on the top-right corner.
![edit-messaging-sessions-records](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/edit-messaging-sessions-records-9.png)

    <Note>For new chat, open any **Messaging Session Record Page**, and then click **Edit Page**.</Note>

    ![editing-messaging-session-record-page](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/editing-messaging-session-record-page-10.png)

4. Under **Custom - Managed** components, find the **OmniChannel_AgentAssistChat** component.
![omnichannel-agentassist-chat](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/omnichannel-agentassist-chat-11.png)

    <Note>For new chat (web messaging), the **Edit Page** appears like the following screenshot:</Note>

    ![edit-page-for-new-chat](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/edit-page-for-new-chat-12.png)

5. Drag and drop the component (search with **OmniChannel_AgentAssistChat**) on the **Record** page.
![component-on-record-page](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/component-on-record-page-13.png)

6. Click **Save**.
7. Click **Activate** and activate the page as per your requirement.
8. Go back to the **Service Console** and initiate another chat from the customer-facing or the demo web page.

## Test your Solution

1. Go to the **Service Console** by searching for it in the **App Launcher**.
![app-launcher-service-console](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/app-launcher-service-console-14.png)

2. In the utility bar, open **Omni-Channel** and set the presence status to **Available-Messaging** or **Available-Chat**, based on the incoming request type.
![omnichannel-presence-status](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/omnichannel-presence-status-15.png)

3. When Salesforce receives an incoming chat or messaging request, a notification appears. Click the **tick** **mark** icon to accept it.
![accept-chat-or-messaging-request](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/accept-chat-or-messaging-request-16.png)

4. After you accept the incoming messaging request, the Agent AI widget appears on the **Messaging Session Record Page** or wherever you placed it in the [previous step](#add-component-to-the-page).
![agent-ai-widget](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/agent-ai-widget-17.png)

5. The Agent AI widget continues to monitor customer and agent messages and provides suggestions based on customer utterances.
6. When the agent clicks the **End Chat** button or the customer disconnects the session, a **summary** pop-up appears on the Agent AI widget.
![widget-summary](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/widget-summary-18.png)

7. Click the **Submit** button to save the summary in one of the custom objects in Salesforce.  \
To access these summaries, search for **Kore NonCTI AgentAI Conversation Summary** in the **App Launcher**.
![kore-noncti-agentai-conversation-summary](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/kore-noncti-agentai-conversation-summary-19.png)

    <Note>The Agent AI widget now supports custom data. All custom field values available in the Messaging Session object in Salesforce are included as custom data in the Agent AI widget. Go through [this doc](/agentai/agent-experience/access-custom-data-in-agent-ai) to access custom data in Agent AI bot.</Note>

    ![custom-data-in-agentai-bot](/ai-for-service/agentai/integration/salesforce-omnichannel-chat-messaging/images/custom-data-in-agentai-bot-20.png)
