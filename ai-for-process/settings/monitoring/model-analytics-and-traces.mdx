# Model Analytics and Traces

The **Model Analytics Dashboard** provides aggregate performance metrics for all model types in your account. **Model Traces** offers run-level analysis for each model request, including inputs, outputs, and metadata. Use the dashboard for trend analysis and the traces view for troubleshooting individual runs.

## Model Analytics Dashboard

The **Model Analytics Dashboard** tracks performance for [fine-tuned](/ai-for-process/models/fine-tune-models/create-a-fine-tuned-model.md), [open-source](/ai-for-process/models/open-source-models/configure-your-open-source-model.md), and [external models](/ai-for-process/models/external-models/add-an-external-model-using-easy-integration.md) in your account. It measures:

- Model latency and response times.
- Request success and failure rates.
- Model scaling patterns and usage.
- Credit consumption for deployments and fine-tuning.

### Access the Dashboard

1. Log in. In the AI for Process top menu, click **Settings**.

   <img src="/ai-for-process/settings/monitoring/images/aip-settings-access.png" alt="access settings" title="access settings"/>

2. On the left menu, select **Monitoring** > **Analytics**.
3. Click the **Model Analytics** tab.

The dashboard loads with data for the last 7 days by default. To load the latest data at any time, click the **Refresh** icon.

<img src="/ai-for-process/settings/monitoring/images/model-analytics-dashboard-refresh.png" alt="dashboard refresh" title="dashboard refresh"/>

### Key Metrics

The dashboard displays account-level and model-specific metrics.

#### Account-Level Metrics

These metrics summarize credit usage and model counts across your account for the selected period.

- **Credits consumed in deployment**: Credits deducted from your account for deploying models.
- **Credits consumed in fine-tuning**: Credits deducted from your account for fine-tuning models.
- **Number of Deployed Models**: Total models deployed in your account.
- **Number of Fine-tuned Models**: Total models fine-tuned in your account.

<Note>Hover over the **i** icon to view a summary for each metric.</Note>

<img src="/ai-for-process/settings/monitoring/images/hover-over-icon.png" alt="hover over icon" title="hover over icon"/>

#### Model-Specific Metrics

**Fine-tuned and Open-source models**

- **Model Latency**: Per-request latency for a 24-hour or 7-day period. For date ranges longer than 7 days, average daily latency is shown. See [Model Latency](#model-latency).
- **Requests**: Success and failure trends, total requests, and average credits per request. See [Requests](#requests).
- **Model Scaling and Usage**: Number of replicas and hardware configuration during deployment. See [Model Scaling and Usage](#model-scaling-and-usage).

**External models**

- **Tokens**: Bar chart of input and output tokens for the selected period. See [Tokens](#tokens).
- **Model Latency**: Same as above for fine-tuned and open-source models.
- **Requests**: Same as above, but without average credits per request.

### Filter Dashboard Data

Combine [Global Timeline Filters](#global-timeline-filters) and [Model Performance Filters](#model-performance-filters) to generate targeted analytics.

#### Global Timeline Filters

The filter options are identical to those on the Workflows Analytics Dashboard. See [Global Timeline Filters](/ai-for-process/settings/monitoring/workflows-analytics-dashboard.mdx#global-timeline-filters) for available options and usage.

#### Model Performance Filters

These filters apply only to [model-specific metrics](#model-specific-metrics) and work alongside a global timeline selection. The available filters depend on the selected model type.

<Note>If multiple deployments exist for the same model name, they are listed with their respective deployment timestamps.</Note>

**Fine-tuned and Open-source models**

- **Select Model (Model Name)**: Select the deployed model.
- **Select Deployment**: Select the model's deployment name.
- **Select Filter (version)**: Select the deployment version.

  <Note>
  - If no versions exist for a model, the dropdown is empty.
  - By default, data for all model versions is shown unless you select a specific version.
  </Note>

  <img src="/ai-for-process/settings/monitoring/images/fine-tuned-model-filters.png" alt="model name and version filter" title="model name and version filter"/>

**External models**

- **Select Model (Model Name)**: Select the deployed model.
- **Select Connection**: Select the third-party service connection name.

  <img src="/ai-for-process/settings/monitoring/images/external-model-filters.png" alt="model name and version filter" title="model name and version filter"/>

### Performance Widgets

To load data in the widgets:

1. Select a date or date range using the [Global Timeline Filters](#global-timeline-filters).
2. Select the model type tab: **Fine-tuned**, **Open-source**, or **External**.
3. For fine-tuned or open-source models, select the model name, and optionally the deployment name and version. For external models, select the model name and connection name.

The widgets update automatically. Selecting a single day shows hourly data; selecting a date range shows daily data.

<Warning>
- Widgets show data only for periods the model was deployed. For example, if a model was active for 4 days and undeployed for 3 days in a 7-day view, only the 4 active days appear.
- The graph shows curves only when requests are processed. No data points appear for hours or days with no activity.
</Warning>

#### Model Latency

Available for **Fine-tuned**, **Open-source**, and **External** models. This widget shows a line graph of the selected model version's latency during the selected period.

- The Y-axis shows **Latency** in milliseconds and scales automatically.
- The X-axis shows the date.
- A blue line shows per-request latency for periods of 24 hours or 7 days. For date ranges longer than 7 days, the graph shows average daily latency.

Hover over a data point to view the latency for a specific request (for 24-hour or 7-day periods) or the average latency for a specific date and time (for longer ranges).

<img src="/ai-for-process/settings/monitoring/images/hover-over-info.png" alt="hover over info" title="hover over info"/>

Use this widget to:

- Identify peak usage and low-efficiency periods by tracking latency over time.
- Detect sudden latency spikes that indicate performance issues.
- Compare deployed model versions to identify the best-performing option.
- Correlate latency trends with credit usage for resource optimization.

#### Requests

Available for **Fine-tuned**, **Open-source**, and **External** models. This widget shows successful and failed requests as two line graphs for the selected period.

- The Y-axis shows **Number of Requests** and scales automatically.
- The X-axis shows the date.
- Successful requests are displayed in green; failed requests in red.
- **Total Requests**: The combined count of successful and failed requests for the selected period.
- **Avg. credits per request**: Total credits consumed divided by total responses generated.

<Note>Avg. credits per request is not shown for external models.</Note>

Hover over a data point to view the count of successful or failed requests at that specific date and time, and the total count for the full selected period.

<img src="/ai-for-process/settings/monitoring/images/requests-hover-info.png" alt="requests hover info" title="requests hover info"/>

Use this widget to:

- Monitor success versus failure rates and identify failure patterns.
- Detect sudden spikes in failures using the hourly view.
- Compare request performance across model versions.

#### Model Scaling and Usage

Available for **Fine-tuned** and **Open-source** models only. This widget shows a step graph of the number of replicas with a specific hardware configuration deployed for the selected model version.

- The Y-axis shows **Number of Replicas** and adjusts based on deployed replicas.
- The X-axis shows the date.
- Each upward step represents additional replicas being generated. Each downward step represents replicas being undeployed.

<Note>**Multiple Hardware** is shown when replicas with more than one hardware configuration are deployed across a date range.</Note>

Hover over a data point to view the timestamp, deployment version name, replica count, and hardware configuration at that point.

<img src="/ai-for-process/settings/monitoring/images/model-scaling-hover-info.png" alt="model scaling hover info" title="model scaling hover info"/>

**Maximum replicas limit**

For a single-day selection, the widget displays a red line indicating the maximum number of replicas configured during model deployment.

<img src="/ai-for-process/settings/monitoring/images/max-replicas.png" alt="max replicas" title="max replicas"/>

Use this widget to:

- Track replica deployments over time to identify usage spikes and optimize hardware consumption.
- Spot inefficiencies in replica usage and reduce unnecessary deployment costs.
- Monitor scaling against your account's replica limits to stay within subscription thresholds.

#### Tokens

Available for **External** models only. This widget shows a stacked bar graph of input and output tokens for the selected period.

- The Y-axis shows **Number of Tokens** (in thousands, *K*) and scales automatically.
- The X-axis shows the date.
- Input and output tokens are shown in two shades in a stacked bar, displaying them as parts of the total tokens used.
- The widget shows the total sum of input and output tokens.

<img src="/ai-for-process/settings/monitoring/images/input-output-tokens.png" alt="input output tokens" title="input output tokens"/>

Hover over a data point to view the timestamp and the input and output token counts at that point.

<img src="/ai-for-process/settings/monitoring/images/tokens-hover-info.png" alt="tokens hover info" title="tokens hover info"/>

Use this widget to:

- Assess model efficiency by comparing input versus output token counts.
- Track token-based credit consumption and optimize usage.
- Identify processing bottlenecks and usage patterns over time.

## Model Traces

**Model Traces** provides run-level analysis of any model deployed or integrated in your account. Use it to review request inputs, generated outputs, response times, and run metadata. It supports filtering, searching, and exporting data for troubleshooting and performance investigation.

### Access Model Traces

1. Log in. In the AI for Process top menu, click **Settings**.

   <img src="/ai-for-process/settings/monitoring/images/aip-settings-access.png" alt="access settings" title="access settings"/>

2. On the left menu, select **Monitoring** > **Model Traces**.
3. If this is your first time using the feature, select a model from the dropdown menu.

   <img src="/ai-for-process/settings/monitoring/images/get-started-model-traces.png" alt="get started with model traces" title="get started with model traces"/>

The feature loads with data for the last 30 days by default. On return visits, it preloads data from your previous model selection for the same 30-day default.

<img src="/ai-for-process/settings/monitoring/images/preload-model-traces-data.png" alt="preload model traces data" title="preload model traces data"/>

### Select a Model and Time Period

Model Traces displays data across all deployments for the selected model. For open-source and fine-tuned models, you can filter by deployment name. For external models, data is shown by connection name.

**Model Name filter**

Select the model to monitor using the **Model Name** filter:

- For open-source and fine-tuned models, select from available deployment names, including version options.

  <img src="/ai-for-process/settings/monitoring/images/traces-os.png" alt="model name filter" title="model name filter"/>

- For commercial models, select the default connection linked to the model.

  <img src="/ai-for-process/settings/monitoring/images/traces-external.png" alt="external filter" title="external filter"/>

**Time-based filter**

Use the time selection filter to scope data to a specific period for targeted analysis or debugging.

<Note>
- **Last 30 Days** is the default selection.
- Data appears only if the model processed requests during the selected period.
</Note>

Available time periods:

- **All Time**: All runs since the account was created.
- **Today**: Data for the current day.
- **Yesterday**: Data for the previous day.
- **This Week**: Data for all days in the current week.
- **This Month**: Data for all days in the current month.
- **Last Month**: Data for all days in the previous month.
- **Last 30 Days**: Data for the past 30 days from today (default).
- **Last 90 Days**: Data for the past 90 days from today.
- **This Year**: Data for all days in the current year.
- **Last Year**: Data for all days in the previous year.

To set the time range:

1. Click the time selection button (displays **Last 30 Days** by default).

   <img src="/ai-for-process/settings/monitoring/images/time-selection-button.png" alt="time selection button" title="time selection button"/>

2. Select a period from the left panel, or select a specific date on the calendar widget.
3. Click **Apply**.

   <img src="/ai-for-process/settings/monitoring/images/select-and-apply.png" alt="select date" title="select date"/>

The selected date range is shown at the bottom of the calendar widget. You can navigate months using the forward and backward arrows, or select a specific month or year from the dropdown lists.

<img src="/ai-for-process/settings/monitoring/images/date-range-display.png" alt="date range display" title="date range display"/>

<img src="/ai-for-process/settings/monitoring/images/calendar-widget.png" alt="calendar widget" title="calendar widget"/>

To set a custom date range, click a start date on the calendar. The current day is set as the end date by default.

<img src="/ai-for-process/settings/monitoring/images/default-calendar-selection.png" alt="default selection" title="default selection"/>

### Performance Metrics Summary

The metrics summary at the top of the page shows key performance data for the selected model and period.

- **Total Requests**: Total runs serviced by the model since deployment. Reflects processing speed and efficiency.
- **Response Time**: P90 and P99 thresholds — the response times below which 90% and 99% of requests fall, respectively. Lower values indicate consistent speed; higher values indicate potential issues.
- **Failure Rate**: Percentage of requests that failed or were not serviced out of total requests. For example, 5 failed requests out of 100 equals a 5% failure rate.
- **Hosting Credits**: Credits consumed by the deployed model based on usage, for comparison against actual utilization.

<Note>Hosting Credits apply only to AI for Process open-source and fine-tuned models. This metric is not shown for external models.</Note>

<img src="/ai-for-process/settings/monitoring/images/hosting-credits.png" alt="hosting credits" title="hosting credits"/>

### Model Traces Table

The table displays all runs for the selected model, sorted from the latest to the oldest execution date. It includes data from the first execution onward — whether from deployment (open-source and fine-tuned models) or integration (external models).

Successful requests are marked with a green check icon; failed requests with a red alert icon.

<img src="/ai-for-process/settings/monitoring/images/success-failed-requests.png" alt="success and failed requests" title="success and failed requests"/>

The table includes the following columns:

- **Status**: Success or failure icon for the run.
- **Request ID**: Unique identifier for the run record.
- **Response Time**: Time taken by the model to respond.
- **Deployment Version**: Model version deployed in your account.
- **Source Type**: The type of source that initiated the request.
- **Source**: The specific source name that sent the request.
- **Executed on**: Run execution timestamp. Click the **Sort** icon in this column to reorder records by ascending or descending date.
- **Input**: Input text provided for the run.
- **Output**: Output text or response generated by the model.

<img src="/ai-for-process/settings/monitoring/images/model-traces-sort.png" alt="model traces sort" title="model traces sort"/>

To view the detailed trace for a run, click its row in the table.

<img src="/ai-for-process/settings/monitoring/images/detailed-trace-information.png" alt="click traces record" title="click traces record"/>

### Customize the Table View

Use the **Visibility Filter** to add or remove columns from the table. Toggle a column on to show it and off to hide it.

<img src="/ai-for-process/settings/monitoring/images/visibility-icon.png" alt="visibility icon" title="visibility icon"/>

<img src="/ai-for-process/settings/monitoring/images/visibility-filter.png" alt="visibility filters" title="visibility filters"/>

<Note>
- All columns are visible by default.
- You can adjust visibility for 8 columns for open-source and fine-tuned models, and 7 columns for other model types.
- The **Deployment Type** filter is available only for open-source and fine-tuned models.
</Note>

### Search Records

Enter a **Request ID** or a string value in the **Search** field to locate specific runs.

<img src="/ai-for-process/settings/monitoring/images/search-record-model-traces.png" alt="search model traces record" title="search model traces record"/>

### Filter by Column

Apply column filters to narrow down the records displayed. You can filter by a single column or combine multiple columns using AND/OR operators.

**Add a filter**

1. Navigate to **Model Traces**.
2. Click the **Filter** icon.
3. Click **+ Add Filter**.

   <img src="/ai-for-process/settings/monitoring/images/click-add-filter.png" alt="add filter" title="add filter"/>

4. In the **Filter By** window, select values from the **Select Column**, **Operator**, and **Value** dropdown lists. For some columns, you must enter the value manually.

   <img src="/ai-for-process/settings/monitoring/images/filter-selection-model-traces.png" alt="filter selection" title="filter selection"/>

5. Click **Apply**.

   <img src="/ai-for-process/settings/monitoring/images/apply-filter-model-traces.png" alt="apply filter" title="apply filter"/>

The table displays matching records. The number of active filters is shown on the **Filter** icon.

<img src="/ai-for-process/settings/monitoring/images/number-of-filters.png" alt="number of filters" title="number of filters"/>

To clear all filters, click **Clear All**.

<img src="/ai-for-process/settings/monitoring/images/clear-all-link.png" alt="clear all" title="clear all"/>

The following table summarizes available filter columns, supported operators, and values.

| Column | Description | Operators | Input Type | Value |
|--------|-------------|-----------|------------|-------|
| **Status** | Run status | Is Equals To, Is Not Equals To | List | Failed, Success |
| **Request ID** | Unique run identifier | Is Equals To, Is Not Equals To, Contains | Enter manually | Any value |
| **Response Time** | Model response time for the request | Is Equals To, Is Not Equals To, Is Greater Than, Is Less Than, Is Greater Than Equals To, Is Less Than Equals To | Enter manually | `m:s:ms` (minutes:seconds:milliseconds) |
| **Deployment Version** | Model version deployed for the run | Is Equals To, Is Not Equals To, Contains | Enter manually | Any value |
| **Source Type** | Source type that sent the request | Is Equals To, Is Not Equals To, Contains | List | [Workflow](/ai-for-process/workflows/overview.md), [Prompts](/prompts/using-prompt-studio/), [API Key](/ai-for-process/models/open-source-models/generate-an-api-key-open-source/) |
| **Source** | Specific request origin | Is Equals To, Is Not Equals To, Contains | List | Custom user-defined values |

**Add multiple filters**

Combine filters using AND or OR operators for targeted, multi-level filtering.

<Warning>
- AND and OR operators cannot be combined in a single filter configuration. Choose one operator and apply it consistently across all filter steps.
- **AND**: All conditions must be met for a record to appear.
- **OR**: Any condition being met includes the record.
- To remove a filter step, click the **Delete** icon.
</Warning>

<img src="/ai-for-process/settings/monitoring/images/delete-filter-icon.png" alt="delete filter" title="delete filter"/>

To add multiple filters:

1. Follow steps 1–3 from [Add a filter](#add-a-filter) above.
2. In the **Filter By** window, select the **AND** or **OR** operator tab.

   <img src="/ai-for-process/settings/monitoring/images/select-operator-filter.png" alt="select operator" title="select operator"/>

3. Follow steps 4–5 from [Add a filter](#add-a-filter) above.

### Export Data

Click **Export** to generate a CSV file of model traces records based on the selected date range and filters. All eight table columns are included in the export regardless of the current visibility filter settings.

<img src="/ai-for-process/settings/monitoring/images/click-export.png" alt="export" title="export"/>

The UI shows export progress while the file is being prepared.

<img src="/ai-for-process/settings/monitoring/images/export-progress-status.png" alt="export progress" title="export progress"/>

Once downloaded, a confirmation message appears.

<img src="/ai-for-process/settings/monitoring/images/export-model-traces-success.png" alt="export success" title="export success"/>

If an error occurs during export, an error notification is displayed.

<img src="/ai-for-process/settings/monitoring/images/export-failed-notification.png" alt="export error" title="export error"/>

The file is saved automatically using the format `modelname_traces_data` — for example, `GPT4_traces_data`.

<img src="/ai-for-process/settings/monitoring/images/export-schema.png" alt="export schema" title="export schema"/>

<Note>
- Each user's export runs independently. One user's cancellation or adjustment doesn't affect another user's export.
- Users can cancel an ongoing export operation.
</Note>

### Trace Details

Click a run record in the table to open the **Traces** window, which shows the Request ID and the following panels.

#### Input and Output Panels

- **Input**: The request text provided to the model using input tokens.
- **Output**: The text output or response generated by the model using output tokens.

Plain text is the default display format. Enable **JSON mode** to view the complete request or response payload in the code editor, including additional keys not visible in plain text.

<Note>In the JSON code editor, the model name shown includes the deployment version for open-source and fine-tuned models, and the connection name for external models.</Note>

<img src="/ai-for-process/settings/monitoring/images/json-editor-model-traces.png" alt="json editor" title="json editor"/>

The text and JSON code in the editor are read-only. To use the content for debugging, click the **Copy** icon to copy it to your clipboard.

<img src="/ai-for-process/settings/monitoring/images/copy-input.png" alt="copy input" title="copy input"/>

#### Metadata Panel

The metadata panel shows run-specific information to support performance analysis.

**Fine-tuned and Open-source models**

- **Request ID**: Unique identifier for the request.
- **Base model**: The platform-hosted or imported model that handled the request.
- **Deployment name**: The deployment name of the model.
- **Deployment version**: Version deployed for the run.
- **Response time**: Time taken to generate a response.
- **Input tokens**: Token count in the request input.
- **Output tokens**: Token count in the model's response.
- **Executed on**: Date and time of the run.
- **Source type**: Type of source that sent the request.
- **Source**: Specific origin of the request.
- **User ID**: Identifier of the user who initiated the request.

<img src="/ai-for-process/settings/monitoring/images/os-metadata.png" alt="metadata" title="metadata"/>

**External models**

Includes all fields above (excluding **Deployment name** and **Deployment version**), plus:

- **Connection name**: The deployed connection name for the model.

<img src="/ai-for-process/settings/monitoring/images/external-metadata.png" alt="external model metadata" title="external model metadata"/>
