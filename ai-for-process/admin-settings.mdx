The Settings console is the centralized management interface for account administrators. Use it to configure users, roles, workspaces, integrations, security, and monitoring across AI for Process.

---

## Workspaces

A workspace is a logical grouping of workflows and configurations — typically organized by team or project. Every account starts with a default workspace assigned automatically.

### Workspace Switcher

The active workspace is shown in the top-right corner of the platform. Click it to switch workspaces or browse all available ones.

| Feature | Description |
|---------|-------------|
| **Default workspace** | Mark any workspace as default — it loads automatically at login |
| **Search and browse** | Find workspaces via search or pagination |
| **Favorite workspace** | Mark up to three workspaces as favorites for quick access|
| **Request access** | Request access to workspaces listed under **Other Workspaces** |
| **Role visibility** | See your role and member count per workspace |

Switching workspaces refreshes the workflows, permissions, settings, models, and deployment environments shown in the UI.


## Settings Overview


| Feature | Description |
|---|---|
| [**User Management**](/ai-for-process/settings/user-management) | Invite and manage users individually or in bulk, assign roles, and unlock accounts. Sync users automatically from Active Directory on a scheduled cadence. |
| [**Roles and Permissions**](/ai-for-process/settings/role-management) | Control what users can do across the platform. View system roles, create custom roles, and set access levels per module. |
| [**Integrations**](/ai-for-process/settings/integrations) | Connect to 130+ pre-built third-party integrations organized by category. Create secure connections using the required authentication method and use them in workflows via the Integration node. |
| [**Custom Scripts**](/ai-for-process/settings/manage-custom-scripts) | Import and manage reusable scripts that run in isolated containers. Upload, configure, deploy, and invoke scripts from the Function node in workflows. |
| [**Manage OCR Models**](/ai-for-process/settings/manage-ocr) | Configure and manage document-processing engines for text extraction and layout understanding. Deploy Docling models or connect Azure Doc Intelligence for use in the Doc Intelligence Node. |
| [**Browser Automation**](/ai-for-process/settings/deployment) | Deploy automation pods and create browser-based experiments that execute step-by-step workflows across web applications. Publish automations for use in the Browser Automation node within workflows. |
| [**Guardrails**](/ai-for-process/guardrails) | View and manage pre-deployed guardrail models that scan prompt inputs and outputs across all workflows, including Anonymize, Ban Topics, Prompt Injection, Toxicity, Bias Detection, and Relevance. |
| [**Monitoring**](/ai-for-process/settings/monitoring-overview) | Track user activity, workflow and model performance, run-level metrics, and script execution logs across dedicated dashboards. All dashboards support date range filtering and search. |
| [**Security and Control**](/ai-for-process/settings/security-control) | Configure SSO for centralized authentication, OAuth2 authorization profiles for reuse across workflows and models, and API-scoped keys to restrict access to specific endpoints. |
| [**Billing and Usage**](/ai-for-process/settings/billing-and-usage) | Monitor credit consumption and usage trends across workflows, models, guardrails, OCR, custom scripts, and browser automation deployments. |
