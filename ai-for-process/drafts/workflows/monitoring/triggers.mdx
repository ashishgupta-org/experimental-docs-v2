The Triggers page shows all configured triggers for a workflow — their configuration, timing, and recent activity. Use it to monitor trigger health, identify inactive triggers, and keep workflows running as expected.

<Note>
Only **Deployed** workflows can have active triggers. Inactive triggers display an error message with recommended actions.
</Note>

---

## Access the Triggers Page

1. Log in to AI for Process and go to **Workflows**.
2. Select a deployed workflow.
3. In the left navigation pane, click **Monitoring** > **Triggers**.
4. Click any trigger row to expand its details.

---

## Event-Based Trigger Details

| Field | Description |
|-------|-------------|
| **Provider** | The service generating the trigger (for example, a webhook provider). |
| **Trigger Name** | Unique name of the trigger. |
| **Description** | Summary of what the trigger does. |
| **Setup Time** | When the trigger was configured. |
| **Last Run** | The last time the trigger executed the workflow. |



## Schedule-Based Trigger Details

| Field | Description |
|-------|-------------|
| **Frequency** | How often the trigger fires (for example, hourly or daily). |
| **Description** | Summary of the scheduled trigger. |
| **Setup Time** | When the schedule was configured. |
| **Start Time / End Time** | The execution window of the trigger. |
| **Time Zone** | Time zone used for scheduling. |
| **Last Run / Next Run** | Execution history and the next scheduled run. |



## Best Practices

- Check the Triggers page regularly to confirm all triggers are active.
- Expand trigger details selectively to keep the view uncluttered.
- Act on inactive trigger alerts promptly to prevent workflow failures.
