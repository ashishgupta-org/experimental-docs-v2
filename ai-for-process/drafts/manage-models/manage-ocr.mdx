The **Manage OCR Models** page is where you configure and maintain the document-processing engines used across your workflows. Engines such as **Docling** and **Azure Doc Intelligence** power the Doc Intelligence node, enabling text extraction, layout understanding, and structured data output.

Use this page to enable engines, deploy models, and manage OCR configurations.

---

## Docling Models

Docling models provide AI-powered document extraction, including text, layout, and structured data. You can have multiple deployments of Docling models. Configure Docling here before it appears as an available engine in the Doc Intelligence node.

Select the **Docling** tab to deploy new models or manage existing deployments.

### Deploy a Docling Model

1. Go to **Settings** > **Manage Models** > **OCR**.
2. Click **Deploy a Model**.
3. Under **General Details**, enter a unique deployment name and an optional description.
   - English (`en`) is selected by default. You can add one additional language to improve model performance.
4. Under **Resource Allocation**, configure the following:
   - Minimum replicas
   - Maximum replicas
   - Auto-scaling threshold
   - Hardware type
5. Review the configuration details.
6. Accept the terms and conditions, then click **Deploy**.

Once deployment is complete, the model appears in the deployed list and becomes available in the Doc Intelligence node.

![Manage Docling Models](/ai-for-process/drafts/manage-models/images/ocr_manage_docling.png)

### View Deployment Details

Click any deployment row to open its details. The following tabs are available:

| Tab | Description |
|-----|-------------|
| **Overview** | Shows deployment status and key configuration: name, selected language, scaling parameters, and hardware. |
| **Deployment History** | Lists all past and current versions with status, timestamps, and ownership. Expand a row to view lifecycle events such as deployments and undeployments, configuration changes, and duration. |
| **Configurations** | Provides options to undeploy an active deployment or delete an undeployed configuration. |

![Overview Tab](/ai-for-process/drafts/manage-models/images/ocr_overview.png)

![Deployment History Tab](/ai-for-process/drafts/manage-models/images/ocr_deployment_history.png)

![Configurations Tab](/ai-for-process/drafts/manage-models/images/ocr_configurations.png)

### Undeploy a Deployment

On the **Configurations** tab, click **Undeploy**. Once undeployed, the model is no longer available to workflows.

### Delete a Deployment

<Warning>You can only delete a deployment after it is undeployed. Deletion is disabled while a deployment is active.</Warning>

On the **Configurations** tab, click **Delete** to permanently remove the configuration.

## Azure Doc Intelligence

The **Azure Doc Intelligence** tab lets you connect external Azure document-processing models for use in the Doc Intelligence node. You can add multiple Azure connections, manage them, and control which ones are active. Configure Azure here before it appears as an engine option in the Doc Intelligence node.

### Add a Connection

1. Go to **Settings** > **Manage Models** > **OCR** > **Azure Doc**.
2. Click **Add a Connection**.
3. Enter the following details:
   - **Connection Name** — must be unique.
   - **Base URL**
   - **API Key**
4. The system validates the API key and checks for duplicate names.
5. After validation succeeds, click **Save**.

A confirmation message appears and the connection is added to the available list.

### Manage Connections

Use the following actions on any connection in the list:

| Action | Description |
|--------|-------------|
| **Enable or Disable** | Toggle the switch next to a connection to enable or disable it instantly. |
| **Delete** | Click the delete icon and confirm to permanently remove the connection. |
