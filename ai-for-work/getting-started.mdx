## Platform Access

Users log in to AI for Work at [https://work.kore.ai/login/](https://work.kore.ai/login/).

Admins can configure [user enrollments](/ai-for-work/administration/user-management) to manage who can access the platform. Users can also sign in using:

- Google Workspace or Microsoft Office profiles.
- Your organization's SSO after configuring [Single Sign-on](/ai-for-work/administration/security-and-control#single-sign-on-sso).

**Supported browsers**: Google Chrome, Microsoft Edge, Apple Safari, and Mozilla Firefox (latest versions).

### Phase 1: Foundation

Get your account and access controls ready.

| Step | Description |
|------|-------------|
| 1. **Sign Up** | Contact the [Sales team](https://kore.ai/contact-us/) to create a new AI for Work account. |
| 2. **Configure User Enrollments** | Set up user enrollments to manage account access and invitations. [Learn more вЖТ](/ai-for-work/administration/user-management#user-enrollment) |
| 3. **Configure SSO** | Set up Single Sign-On for easy access using your identity provider. [Learn more вЖТ](/ai-for-work/administration/security-and-control#single-sign-on-sso) |
| 4. **Add Service Accounts** | Add Google, Microsoft, or LDAP service accounts to get user profiles and distribution lists. [Learn more вЖТ](/ai-for-work/administration/security-and-control#service-accounts) |


### Phase 2: Configuration

Configure models, rate limits, and enterprise knowledge.

| Step | Description |
|------|-------------|
| 1. **Configure LLM and Embedding Models** | Integrate with a pre-built or custom LLM. Pre-built integrations include Azure OpenAI, OpenAI, and Anthropic. Use the bring-your-own (BYO) framework for external or self-hosted models. [Learn more вЖТ](/ai-for-work/llm-configuration) |
| 2. **Configure Rate Limits** | Set query rate limits per user category. Simple queries (small talk) use 1 point; advanced queries (knowledge integration, context) use 3 points. Configure limits separately for Moderate Users and Power Users. [Learn more вЖТ](/ai-for-work/assistant-configurations#rate-limits) |
| 3. **Configure Enterprise Knowledge** | Set up the Enterprise Knowledge Search AI app вАФ a pre-configured fallback that generates responses when no specific agent matches the user's intent. [Learn more вЖТ](/ai-for-work/custom-agents/enterprise-knowledge) |
| 4. **Configure Work Orchestrator Agent** |  Orchestrator AgentSet up Work вАУ Agentic Search to enable intelligent search and orchestration across your organization's knowledge sources. This includes creating the agentic application, configuring language models, setting up MCP tools, and deploying the agent workflow. [Learn more вЖТ](/ai-for-work/custom-agents/enterprise-knowledge#work-agentic-search) |


### Phase 3: Deployment

Publish and configure agents for your users.

| Step | Description |
|------|-------------|
| 1. **Publish Pre-built Agents** | Publish pre-built agents (content summarization, email drafting, translation, and more) from the Agent Store with minimal setup. [Learn more вЖТ](/ai-for-work/pre-built-agents) |
| 2. **Configure Custom Agents and Applications** | Build agents and applications that collect user input, retrieve data, connect to APIs, and execute complex tasks. [Learn more вЖТ](/ai-for-work/custom-agents) |


### Phase 4: Go Live

Invite your users to the platform.

| Step | Description |
|------|-------------|
| 1. **Invite Users** | Share access with your team. The AI for Work Assistant gives employees a single entry point to knowledge across systems and all published AI agents. [Learn more вЖТ](/ai-for-work/administration/user-management#user-enrollment) |
| 2. **Help Users Get Started** | Share guidance on how to use the AI for Work Assistant. [Learn more вЖТ](/ai-for-work/getting-started) |
